Settings

The settings panel gives you maximum control over the ASCII import process, allowing to import a great variety of ASCII files into the TA GUI.

Template

As the same kind of ASCII files needs always the same set of parameters for the import, there will be the possibility both to create/save such sets of parameters and to select them from a popupmenu.

Please note: This feature has not been implemented yet.

General

There is a number of important parameters for the import of ASCII files that are grouped together in the "General" panel.

Separator

The separator used in the ASCII file to separate the fields in a row from each other, such as ",", ";", "\t" (for TAB).

Tip: Leave the field empty for automatic detection.

Header

The number of header lines in the ASCII file.

Dimensionality

Whether it is a full 2D (normally time vs. wavelength) or a 1D dataset (time or wavelength). As with the TA GUI, the x axis is always associated with the time dimension, the y axis with the wavelength dimension, and the z axis with the intensity.

Please note: As of early June 2013, the dimensionality feature has not been thoroughly tested. Therefore, please handle with care and please, do report bugs to the author of the toolbox (either via BugZilla or directly via email).

Transpose

Whether to transpose the data in the ASCII file. Transposing is necessary if the rows are not the time profiles.

Please note: The data get transposed rather late in the import process. Therefore, specifying rows and/or columns for axes refers to the situation as in the file, i.e. before transposing the data.

Axes

As noted above and consistent with the TA GUI, the x axis is always associated with the time dimension, the y axis with the wavelength dimension, and the z axis with the intensity.

For each axis, you can provide information about measure and unit, respectively. Please note that these informations are optional and can easily be changed afterwards via the "Info GUI" in the TA GUI.

Values

For x and y axes, the values can be set in a couple of different ways, the easiest (and default) being the index, meaning that the axes values are simply the indices of the dimensions of the dataset.

Another option is to specify an explicit range for the axes. Here, the importer function assumes equal spacing. Therefore, setting the popup menu to "range" will allow you only to enter the boundaries of the respective axis (in the unit specified with "unit" above).

The third option is to use one row or column of the data, as sometimes the first row/column of the data block is the respective axis. The row/column specified gets removed from the dataset.

Save settings as template

Save a set of import parameters for a given file type to allow easy and fast import. Each template gets a short name (short enough to be sensibly displayed in the popup menu on top of the settings panel). Pressing the Save button will save the parameters as template with the given name.

Please note: This feature has not been implemented yet.
